# 从字节码角度分析 i++ 和 ++i

本篇文章从字节码的角度分析`i++`和`++j`背后的逻辑！！

先给出四个可能会让你很晕～的例子，相信如果搞清楚`i++`和`++j`原理的人可以很快知道每个变量最终的结果

```java
public void add() {
    // problem 1
    int i1 = 10;
    i1++;
    int i2 = 10;
    ++i2;
    
    // problem 2
    int i3 = 10;
    int i4 = i3++;
    int i5 = 10;
    int i6 = ++i5;
    
    // problem 3
    int i7 = 10;
    i7 = i7++;
    int i8 = 10;
    i8 = ++i8;
    
    // problem 4
    int i9 = 10;
    i9 = i9++ + ++i9;
}
```

**<font color='red'>总结：</font>**

如果针对`i`本身的话，都会在原有的基础上`+1`

如果针对表达式`i++`和`++i`的话

- `i++`是先把原来的值推送至栈顶，然后再`+1`，最后把栈顶的值赋值给表达式的左边。所以表达式`i++`是 i + 1 之前的值

- `++i`是先`+1`，然后把新值推送至栈顶，最后把栈顶的值赋值给表达式的左边。所以表达式`++i`是 i + 1 之后的值
